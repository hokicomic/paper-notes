Abstract

End-to-end OCR models, exemplified by DeepSeek OCR, use a large language model as decoder, but as the output sequence lengthens, the accumulated KV cache drives up memory consumption and progressively slows down generation.
以 DeepSeek OCR 為代表的端到端 OCR，會利用大型語言模型作為解碼器來提升辨識能力；但輸出序列越長，KV cache 就越大，進而增加記憶體占用並拖慢生成速度。
We propose Unlimited OCR, a model designed to emulate human parsing working memory. Taking DeepSeek OCR as the baseline, we replace all attention layers in the decoder with Reference Sliding Window Attention (R-SWA), which reduces attention computation costs while maintaining a constant KV cache throughout decoding.
作者提出 Unlimited OCR，企圖模仿人類在長時間抄寫或解析時的工作記憶。它以 DeepSeek OCR 為基線，將解碼器內所有注意力層替換成 R-SWA，使模型在解碼期間維持固定大小的 KV cache，並降低注意力計算成本。
By combining DeepSeek OCR's high-compression encoder with constant KV cache design, Unlimited OCR can transcribe dozens of pages in a single forward pass under a standard maximum length of 32K. R-SWA is claimed to be a general-purpose parsing attention mechanism applicable to OCR, ASR, translation, and similar reference-based tasks.
透過高壓縮率的視覺編碼器與固定 KV cache 設計，Unlimited OCR 宣稱能在 32K 最大長度限制下，一次前向傳播轉錄數十頁文件。作者進一步把 R-SWA 定位為通用的 parsing attention，可延伸到 ASR、翻譯等有固定參考來源的任務。

段落功能

快速建立問題：LLM decoder 讓 OCR 更強，但長輸出造成 KV cache 與速度瓶頸。

邏輯角色

提出本文的主方法與核心承諾：用 R-SWA 讓解碼端記憶體不隨生成長度線性成長。

潛在漏洞

「Unlimited」是命名策略，論文本身後面承認受 32K context 與 prefill 長度限制，不能字面理解為無限。

Introduction

Humans can transcribe hundreds of pages or translate hours-long audio, yet no existing OCR model can parse even tens of pages in a single forward pass. Current models usually process documents page by page, resetting memory each time.
人類可以連續抄寫大量頁面或處理長時間語音，但現有 OCR 模型通常無法在單次前向傳播中解析數十頁文件，只能逐頁處理並在每一步重置記憶。
The for-loop paradigm fragments a coherent long-horizon process into isolated short tasks managed by an external scheduler. It works to some extent, but remains an engineering workaround.
逐頁 for-loop 處理把本來連續的長程任務切碎成一個個短任務，由外部排程器維持流程。這能解決工程需求，但不是模型本身具備長程解析能力。
When copying, humans do not re-scan the entire history; they attend to the source text, recent context, and the next item. This motivates an attention pattern that is neither full attention nor linear attention: reference tokens remain visible, while recent outputs form a bounded working memory.
人類抄寫時不會反覆讀完整歷史，而是看原文、剛寫下的局部上下文與下一個目標。作者因此主張需要一種不同於 full attention 與 linear attention 的模式：參考來源持續可見，輸出歷史只保留近期工作記憶。

段落功能

把 OCR 的技術瓶頸提升成長程解析問題，避免本文看起來只是單一模型優化。

論證技巧

作者用「人類抄書」類比降低 R-SWA 的理解門檻。但這是啟發式比喻，不是認知科學證據。

可學習新知

有些長文件任務不需要完整自回歸歷史；真正不可丟的是 reference source 與近端 progress state。

Method｜R-SWA 與架構

R-SWA attends to all reference tokens—visual tokens and the prompt while limiting output attention to the preceding n tokens, with n defaulting to 128. This keeps the model aware of the full image while tracking progress through a causal sliding window.
R-SWA 讓每個生成 token 都能看見所有參考 token，包括視覺 token 與 prompt；但對已生成輸出，只允許注意到前面固定寬度 n 的 causal sliding window。這讓模型既保有完整圖像資訊，又只用有限工作記憶追蹤進度。
Unlimited OCR adopts DeepSeek OCR as baseline, pairing DeepEncoder with a MoE decoder of 3B total and 500M activated parameters. DeepEncoder can compress a 1024×1024 PDF image to 256 tokens.
模型沿用 DeepSeek OCR 架構：DeepEncoder 負責高壓縮率視覺編碼，MoE 解碼器總參數約 3B、啟動參數約 500M。DeepEncoder 可把 1024×1024 的 PDF 圖像壓縮到 256 個 token，這使多頁 prefill 不至於一開始就爆炸。
For a prefix segment P and decode window Dn(t), R-SWA defines accessible positions as N(t)=P∪Dn(t). The prefix remains globally visible; the decode region slides locally.
在形式化上，R-SWA 把每一步可注意的位置定義為固定 prefix 與動態解碼視窗的聯集。prefix 對所有後續 token 全局可見；生成區只保留最近 n 個 token。
Standard MHA cache grows as CMHA(T)=Lm+T, while R-SWA retains CR-SWA(T)=Lm+min(n,T)≤Lm+n. Thus decode-side cache is upper-bounded by a constant window size.
標準 MHA 的 KV cache 隨生成長度 T 線性成長；R-SWA 則只保留固定 prefix 與最近 n 個輸出 token，因此上限是 Lm+n。這是全文最核心的技術點。

段落功能

把直覺比喻落到可實作的注意力遮罩與 KV cache 管理。

關鍵判斷

R-SWA 的價值取決於任務是否能接受「遠端輸出歷史不可直接注意」。OCR 這類 reference-dominant 任務較合理；開放式長文推理未必適合。

可遷移新知

對任務先分類：固定參考源主導的 parsing 任務，可嘗試 reference-global + output-local attention；需要長程自我推理的任務，不能直接照搬。

Experiments / Evaluation

The authors construct approximately 2 million document OCR samples with a 9:1 ratio of single-page to multi-page data. Multi-page data are synthesized by concatenating single-page data, producing around 200k samples of 2 to 50 pages, all packed into 32K tokens.
作者建構約 200 萬筆文件 OCR 樣本，其中單頁與多頁比例為 9:1；多頁資料由單頁資料串接合成，約 20 萬筆、每筆 2 到 50 頁，並全部打包到 32K token 序列內。
Training starts from the DeepSeek OCR checkpoint, continuing for 4,000 steps with global batch size 256 and maximum sequence length 32K on 8×16 A800 GPUs. The DeepEncoder is frozen and only LLM parameters are trained.
訓練從 DeepSeek OCR checkpoint 繼續，進行 4,000 steps，global batch size 256，最大序列長度 32K，硬體為 8×16 A800 GPU。DeepEncoder 被凍結，只訓練 LLM 參數。
Evaluation uses OmniDocBench v1.5/v1.6 and an in-house long-horizon OCR test set with documents of 2, 5, 10, 20, and 40+ pages.
評測包含 OmniDocBench v1.5/v1.6，以及內部長文件 OCR 測試集，頁數分為 2、5、10、20 與 40+ 頁。

段落功能

說明結果不是零訓練替換 attention，而是基於 DeepSeek OCR 續訓後所得。

證據限制

多頁資料是合成串接，長文件測試集是 in-house。這削弱了「真實世界超長文件」泛化結論的強度。

可操作性

訓練資源不低，但推理端已有公開模型與多種部署路徑，對使用者而言較容易先做 inference 驗證。

Results / Efficiency

On OmniDocBench v1.5, Unlimited OCR reaches 93.23 overall, improving over DeepSeek OCR by +6.22. It also improves formula, table, and reading-order metrics.
在 OmniDocBench v1.5 上，Unlimited OCR 的 overall 達到 93.23，比 DeepSeek OCR 高 6.22，並在公式、表格與閱讀順序等指標上都有提升。
On OmniDocBench v1.6, Unlimited OCR achieves 93.92 overall, slightly above Qianfan-OCR's 93.90 and close to other current SOTA models.
在 OmniDocBench v1.6 上，Unlimited OCR 達到 93.92，略高於 Qianfan-OCR 的 93.90，屬於同級 SOTA 區間；這部分優勢幅度其實很小，不能過度解讀。
For long-horizon OCR, the model maintains strong Distinct-n values up to 40+ pages, with edit distance below 0.11 and Distinct-35 around 97%.
在長文件 OCR 測試中，模型在 40+ 頁仍維持約 97% 的 Distinct-35，edit distance 低於 0.11。這支持它至少能在多頁 one-shot OCR 上維持穩定輸出。
In efficiency analysis, at 6,000 generated tokens DeepSeek OCR's TPS declines, while Unlimited OCR remains stable and is 35% faster.
效率分析顯示，輸出長度增加到 6,000 token 時，DeepSeek OCR 的 TPS 下降，而 Unlimited OCR 維持穩定，速度高出約 35%。這比短文件 benchmark 分數更能體現 R-SWA 的真正價值。

段落功能

用 benchmark 與速度數據支撐「替換注意力不傷準確度，還能提高長輸出效率」。

最強證據

效率曲線與 KV cache 公式互相呼應，能直接支持核心機制。

最弱證據

v1.6 的 SOTA 差距極小；長文件資料非公開，難以確認是否含特定分布偏差。

Limitations / Future Work

The model cannot achieve truly unlimited parsing under a finite context length such as 32K, because it is also constrained by prefill length. As pages accumulate, even compressed visual tokens make the prefill long.
作者承認模型無法在 32K 這種有限 context 下達到真正無限解析，因為 prefill 長度仍受頁數累積限制。即使視覺 token 被高壓縮，多頁文件仍會使 prefill 變長。
Future work includes training longer-context models such as 128K, building a prefill pool, and allowing the model to fetch prefill KV chunks automatically, simulating humans flipping pages.
後續方向包括訓練 128K 等更長 context 模型、建立 prefill pool，並讓模型自動抓取 prefill KV chunks，類似人類翻頁查找來源。

保留懷疑

prefill pool 是有趣方向，但這會引入 retrieval / memory management 問題，不再只是 attention mask 的簡單替換。

本文新知萃取

本文一句話主張

對固定參考來源的長程 parsing 任務，模型不必保留完整生成歷史；只要固定看見 reference tokens，並保留最近輸出的 sliding window，就能在近似不損準確度下控制 KV cache 與速度。

真正的新東西

R-SWA 的任務化設計：reference 全局可見、output 局部可見、KV cache 用 queue 固定容量管理。核心不是「滑動視窗」本身，而是把 reference tokens 排除在狀態轉移外，避免視覺特徵被 recurrent 更新模糊。

改變既有理解

長文件 OCR 不一定要靠更長完整 context 或逐頁 pipeline；若任務的主要資訊在固定來源中，生成歷史可以被壓縮成短期進度狀態。

可遷移做法

可用於 PDF parser、ASR 逐段轉錄、字幕翻譯、文件轉 Markdown、長圖表解析、agent 讀大型 reference files。判斷條件是：reference 是否固定且完整？遠端輸出歷史是否只提供進度，而非必要推理依據？

值得記住的術語

Reference Sliding Window Attention、long-horizon parsing、constant KV cache、prefix global visibility、causal decode window、prefill pool、reference-based task。

可實作 / 可復現任務

1. 用 HF 模型跑多頁 PDF → Markdown。2. 比較一般 OCR pipeline 與 Unlimited OCR 在 10/20/40 頁文件的速度與錯誤。3. 測試小字、表格、公式、雜誌版面等失敗案例。4. 用相同概念設計 ASR 或翻譯的 reference-global attention 實驗。

學習價值與風險

最值得學：把注意力設計回扣到任務結構，而不是單純把 context length 拉長。這對你做論文閱讀系統、agent 工具讀取、多文件解析，都有架構啟發。

最需要懷疑：「Unlimited」容易誤導。真正限制轉移到了 prefill 長度、reference token 數、合成資料分布，以及長文件實測資料是否公開可驗證。

底部論證結構總覽

問題端到端 OCR 使用 LLM decoder 後，長輸出造成 KV cache 與推理速度瓶頸；逐頁 pipeline 又會切斷連續解析狀態。
論點長程 OCR 其實只需要固定 reference + 近期輸出工作記憶，不需要完整生成歷史。
方法R-SWA：所有 reference tokens 全局可見，output tokens 僅保留 causal sliding window；KV cache 變成 Lm+n 的固定上界。
證據OmniDocBench v1.5/v1.6 成績、長文件 in-house 測試、TPS 與 kernel latency 分析。
反駁 / 限制仍受 32K context 與 prefill 長度限制；長文件資料非公開；合成多頁資料可能低估真實文件複雜度。
結論R-SWA 使 reference-based long-horizon parsing 更可行，但「真正無限」仍需 prefill pool 或更高階記憶管理。

一句話版本

Unlimited OCR 的核心不是 OCR 本身，而是提出一種針對固定參考來源任務的長序列解碼記憶策略：reference 永遠可見，生成歷史只保留最近進度。
